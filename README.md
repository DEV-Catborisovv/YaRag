
# YaRag — Интеллектуальная RAG-система визуализации поисковых ответов

### Архитектура и технические решения

Данный проект представляет собой высокопроизводительную RAG-систему (Retrieval-Augmented Generation), разработанную как функциональный аналог сервиса **Яндекс Нейро**. Система предназначена для генерации консолидированных ответов на основе актуальной поисковой выдачи с использованием современных алгоритмов обработки естественного языка.

#### Архитектура компонентов

##### 1. Модуль сегментации текста (Чанкер) — `src/domain/services/chunker.py`
Использует принцип **скользящего окна** (Sliding Window) для обеспечения связности контекста при обработке длинных документов.
* **Безопасная обработка**: Автоматическое принудительное разбиение фрагментов длиннее 4000 символов для соблюдения лимитов API.
* **Перекрытие (Overlap)**: Конфигурация окна (128/64 слов) предотвращает потерю смысла на границах фрагментов.
* **Оптимизация остатков**: Интеллектуальное объединение коротких завершающих фрагментов для минимизации информационного шума.

##### 2. Семантический ранжировщик (Ранкер) — `src/domain/services/ranker.py`
Обеспечивает поиск наиболее релевантной информации на основе векторных представлений.
* **Векторные представления (Embeddings)**: Преобразование текстовых данных в многомерные векторы через Yandex Cloud.
* **Косинусное сходство**: Расчет семантической близости между вектором запроса и векторами фрагментов текста.
* **Гибридный поиск**: Сочетание векторного поиска с классическим алгоритмом **BM25** для точного соответствия ключевым словам (артикулы, названия).
* **Cross-Encoder Reranking**: Финальная пересортировка результатов для достижения максимальной точности.

##### 3. Процессор источников — `src/domain/services/source_processor.py`
Реализует алгоритм **SelectGlue**, оптимизирующий объем передаваемого в модель контекста.
* Система анализирует полные страницы, выделяет наиболее ценные сегменты и «склеивает» их в лаконичное саммари, сохраняя при этом оригинальный логический порядок повествования.

##### 4. Экспертная оценка (LLM-Judge vs BERT)
Выбор YandexGPT в качестве «судьи» на этапе фильтрации был обусловлен необходимостью глубокого понимания контекста. В отличие от быстрых, но поверхностных BERT-моделей, LLM-Judge способен распознавать сложные логические связи и противоречия, что критически важно для минимизации галлюцинаций.

### Соответствие архитектурным принципам Яндекс Нейро

Проект вдохновлен архитектурными решениями, описанными в [техноблоге Яндекса](https://habr.com/ru/companies/yandex/articles/807801/):

1. **Standalone-рефразирование**: Реализовано в `rephrase_query`. Система восстанавливает полный контекст запроса из истории диалога, устраняя анафоры и уточнения.
2. **Селективная очистка контента**: Аналог SelectGlue в `SourceProcessor` позволяет передавать в модель только контент, несущий фактическую нагрузку.
3. **Winner Selection**: На финальном этапе модель выбирает до 5 наиболее авторитетных и дополняющих друг друга источников, следуя принципам разнообразия (Diversity) и верифицируемости.
4. **Citations & Grounding**: Строгие системные инструкции гарантируют, что каждый факт в ответе подтвержден ссылкой на источник, а генерация «из памяти» модели минимизирована.

### Ключевые возможности и оптимизации (обновление от 26.03.2026)

* **Региональная привязка (Geo-Targeting)**: Интегрирован слой извлечения региональных кодов (Geo-ID), что позволяет получать актуальную выдачу для конкретных городов (Екатеринбург, Москва и др.).
* **Доверительный бустинг (EEAT)**: Приоритетное ранжирование авторитетных ресурсов (Wikipedia, РБК, Дзен, профильные агрегаторы). Система автоматически штрафует однообразные источники для повышения охвата.
* **Устойчивый скрапинг (Anti-Bot)**: Использование специализированных API (Scrapfly) для обхода защит (Cloudflare и др.) и получения данных с ресурсов, блокирующих стандартные парсеры.
* **Многоуровневое кэширование**: Интеграция с **Redis** сокращает время повторных запросов в 10 раз и снижает затраты на API Cloud-провайдеров.
* **Автоматизированная оптимизация (DSPy + Optuna)**: Использование фреймворка DSPy (MIPROv2) для автоматической генерации и тестирования системных промптов. Параметры чанкирования динамически оптимизируются под датасет для лучшего охвата (Recall).

### Технические метрики и ограничения

Текущая реализация ориентирована на максимальное соответствие эталонной выдаче Яндекса по метрикам **Overlap** и **Recall**. 

**Особенности оценки:**
* Система демонстрирует лучшие показатели в категориях «Путешествия», «Медиа» и «Общие вопросы», представленных в обучающем датасете.
* В узкоспециализированных коммерческих нишах (например, локальные услуги) может наблюдаться расхождение с эталоном из-за высокой специфичности локальных агрегаторов, что требует дальнейшего расширения `gold_data.json`.

### Быстрый старт

1. Склонируйте репозиторий.
2. Создайте файл `.env` на основе `.env.example` и укажите ваши ключи доступа к Yandex Cloud, XMLRiver и Scrapfly.
3. Запустите систему через Docker Compose:
   ```bash
   docker compose up --build
   ```
4. Откройте `http://localhost` в браузере.

Интерфейс фронтенда автоматически свяжется с бэкенд-сервисом в изолированной докер-сети.
